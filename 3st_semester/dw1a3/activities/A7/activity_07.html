Activity #07: SEO & Chrome Developer Tool

SEO

Além do desenvolvimento de uma página web, é preciso levar em consideração a sua otimização de busca para que os mecanismos de busca sejam preparados para colocar seu site "no mapa". Para isso, há uma cartilha de boas práticas que a nível de desenvolvimento parecem irrelevantes mas que auxiliam diretamente a otimização de busca. Vamos listar algumas delas:

1. Meta Tags

As meta tags são complementos no seu código HTML que indicam ao navegador que o site está preparado para ser rastreado pelos mecanismos de busca.

2. Palavras-chave

Além de descrever os sites para os mecanismos de busca, também é necessário conhecer as melhores palavras para descrever o tema do seu website. Alguns sites são especializados nisso, como o Neilpatel, que auxilia na inserção do melhor vocabulário de descrição, elevando o raiting da sua página.

3. Canonical Tags

Canonical tags são códigos inseridos no HTML para indicar autenticidade do conteúdo produzido, os mecanismos de busca tendem a indicar sites com conteúdos distintos, valorizando a personalidade de informações.

4. Uso adequado dos cabeçalhos para SEO

Se você acha que a definição do "H1", "H2", "H3" não faz a menor diferença uma vez que se define o 'font-size' no CSS, você está completamente enganado. Talvez para os usuários do site isso na faça a diferença, mas para as buscas é primordial. A definição de um simples H1 fará com que o site seja descrito completamente baseado nele, afinal, ele é o título do site, e isso vale para todos, H2 subtítulo, até o H6. Tome cuidado com as atribuições equivocadas.

5. Descrição de imagens

Além de colocar uma descrição adequada nas imagens, isso é fundamental para que o mecanismo de busca saiba do que se trata a imagem inserida no site. Um site sobre cuidadores de macacos devem garantir que as fotos dos macacos apareçam na aba de imagens quando usuários da internet procurarem por ilustrações de primatas.

6. Links internos

Garanta que todas as tags de link tenham URLs de correspondência, deixar as "pontas soltas" pode confundir as buscas.

7. Ferramentas

  1. Yoast SEO: PlugIn WordPress dá dicas de desenvolvimento de SEO no site
  2. SeoQuake: Ferramenta Google para melhorar SEO do site
  3. MozBar: Outra ferramenta para melhorar SEO do site
  4. Neilpatel
  5. Moz
  6. Ahrefs

Chrome Developer Tools

O navegador do Google Chrome disponibiliza várias facilidades para monitorar o funcionamento do site. Com o Developer Tools conseguimos fazer alterações de forma mais rápida até verificar a performance do site, vamos analisar algumas funcionalidades:

1. Elements

Verifica a construção do site, seu elementos, tags e seletores CSS. Pode fazer alterções rápidas facilitando a manutenção. Também pode visualizar os eventos em JavaScript além de ter uma visão em 3D das camadas utilizadas no site (Layers).

2. Style

Nessa aba é possível tratar tudo referente ao CSS e seus estilos inseridos nas páginas. Permitindo também uma alteração rápida.

3. Workspaces

Apesar de fazer alterações rápidas pelo Google Chrome, numa simples atualização da páginas, os progressos são perdidos. Mas com algumas ações é possível que os progressos sejam salvos, simplesmente apontando as alterações para o arquivos salvo na máquina, agora, não será mais necessário uma alteração dobrada.

4. Console

Na aba console, é possível testar execuções de códigos JavaScript, semelhando a um prompt, basta digitar as sintaxes e temos uma resposta imediata de retorno

5. Debug

Permite adicionar breakpoints para que tenha a execução interrompida, com a finalidade de tratamento de erros.

6. Resources

A partir de uma página é possível enxergar todos os componentes carregados dela, como cookies, APIs e outras requisições. Algumas informações são armazenadas para que não haja perdas com uma falha de conexão, chamado "Local Storage".

7. Network

Analisando a perfomance do site, a aba Network mapeará todas as requisições feitas pelo site e a quanidade de dados trafegados, podendo haver análises para uma melhor performance.

8. Audits

Auditorias feitas pelo Chrome no intuito de dar dicas de melhorias para a páginas, possui diversos aspectos de avaliação.

9. Timeline

Testará o tempo de execução das partes do código, mostrando graficamente o quanto cada parte tem excesso de tempo e que deve ser refatorado.

10. Emulator
